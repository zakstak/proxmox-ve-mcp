## 2024-05-24 - [Compact JSON Serialization]
**Learning:** In the context of Model Context Protocol (MCP) server responses intended for LLM consumption, using pretty-printed JSON (`JSON.stringify(data, null, 2)`) is an anti-pattern. It unnecessarily bloats the response string with spaces and newlines. Tests demonstrate that switching to compact JSON (`JSON.stringify(data)`) reduces payload size by ~30% and speeds up serialization by ~22% for moderate datasets (e.g., 100 items). This directly translates to lower token usage and faster processing by the client LLM, without altering the underlying data structure or causing parsing issues.
**Action:** Always default to compact JSON serialization `JSON.stringify(data)` for any programmatic API or tool response that does not require direct human readability in its raw string form. Apply this specifically across all MCP tool handlers returning structured data.
