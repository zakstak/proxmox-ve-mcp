## 2025-03-03 - [Strict Resource Identifier Validation]
**Vulnerability:** Zod string schemas for resource identifiers (node, storage, snapshot names, hostnames, and tasks) lacked strict pattern matching, relying only on `z.string()`. This permitted any characters and introduced a potential path traversal and API injection risk if inputs were passed directly to underlying shell commands or untrusted APIs.
**Learning:** Adding simplistic alphanumeric regex checks initially broke functionality. Proxmox requires specific characters for certain fields: hostnames need dots for FQDNs, UPIDs require colons and '@', and descriptions need to allow whitespace and standard punctuation while blocking common injection vectors like quotes or angle brackets.
**Prevention:** Always implement strict, context-aware regex validation (`z.string().regex()`) tailored to the domain-specific format of the input. Avoid overly broad validation that introduces functional regressions, but ensure dangerous characters are denied for free-text fields.
